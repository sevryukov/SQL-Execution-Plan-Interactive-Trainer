# Hash Join

**Hash Join** строит хеш-таблицу по одному входу (build) и проверяет другой вход (probe) для поиска совпадений.

## Интуиция через план (ASCII)

```text
Build Input (обычно меньший набор)
           |
           v
     +-----------+
     |  BUILD    |----> Хеш-таблица в памяти
     +-----------+
           ^
           |
Probe Input (больший набор)
           |
           v
     +-----------+
     |   PROBE   |----> Выходные строки
     +-----------+
```

## Схема таблиц

```text
Customers
+----------+------------+
|CustomerID|City        |
+----------+------------+
|VINET     |Reims       |
|TOMSP     |München     |
|HANAR     |São Paulo   |
+----------+------------+

Orders
+---------+------------+
|OrderID  |CustomerID  |
+---------+------------+
|10248    |VINET       |
|10249    |TOMSP       |
|10250    |HANAR       |
+---------+------------+
```

## Когда это эффективно

- Один из наборов данных помещается в память.
- Нет полезных индексов для соединения.
- Большие наборы данных без селективных предикатов.
- Требуется соединение по равенству (equi-join).

## Типовой порядок выполнения

1. **Build phase**: читается меньшая таблица, строится хеш-таблица по ключу соединения.
2. **Probe phase**: читается большая таблица, вычисляется хеш и ищутся совпадения.
3. Найденные пары строк возвращаются как результат.

## Преимущества перед Nested Loop

- Лучше масштабируется на больших данных.
- Не требует индексов на соединяемых колонках.
- Эффективен при отсутствии селективности.

> Оптимизатор выбирает Hash Join для больших соединений без подходящих индексов.
