# Nested Loop Join

**Nested Loop Join** сравнивает каждую строку внешнего входа с подходящими строками внутреннего входа.

## Интуиция через план (ASCII)

```text
Внешний вход (фильтрованные Orders)
               |
               v
      +----------------+
      |  NESTED LOOP   |----> Выходные строки
      +----------------+
               ^
               |
Внутренний поиск по ключу (Index Seek по OrderItems.OrderID)
```

## Схема таблиц

```text
Orders
+---------+------------+
|OrderID  |CustomerID  |
+---------+------------+
|10248    |VINET       |
|10249    |TOMSP       |
+---------+------------+

OrderItems
+---------+---------+----------+
|OrderID  |LineNo   |ProductID |
+---------+---------+----------+
|10248    |1        |11        |
|10248    |2        |42        |
|10249    |1        |72        |
+---------+---------+----------+
```

## Когда это эффективно

- Внешний набор данных небольшой.
- На внутренней таблице есть индекс по ключу соединения.
- Ожидается небольшое число совпадений.

## Типовой порядок выполнения

1. Читается одна строка из внешнего источника.
2. По условию выполняется поиск во внутреннем источнике.
3. Найденные строки возвращаются, затем цикл повторяется.
